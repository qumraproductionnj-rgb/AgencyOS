# 🧪 PROMPT 09: كتابة tests فقط

استخدمه عندما تحتاج tests لكود موجود.

```
اكتب tests شاملة للموديول: {{اسم الموديول}}.

التغطية المطلوبة:
- Unit tests: كل function مع 3 حالات على الأقل (happy path + 2 edge cases)
- Integration tests: كل API endpoint
- E2E tests: السيناريوهات الحرجة فقط

الأدوات:
- Backend: Jest + Supertest
- Frontend: Vitest + React Testing Library
- E2E: Playwright

القواعد:
- Test names بصيغة: should_X_when_Y
- استخدم data factories بدل hardcoded data
- نظّف database state بين tests
- لا mocks إلا للخدمات الخارجية (email, SMS, payment)

أعطني خطة قبل الكتابة.
```

# 💾 PROMPT 10: Database migration

```
أحتاج migration جديدة لـ: {{وصف التغيير}}.

اتبع هذه القواعد:
1. الـ migrations لا تحذف عمود إلا بعد deprecation فترة (مرحلتين)
2. كل تعديل schema يصاحبه:
   - migration UP و DOWN
   - تحديث Prisma schema
   - تحديث الـ types
   - تحديث الـ tests
3. تحقق RLS مطبّقة على أي جدول جديد
4. تحقق من الـ indexes الضرورية

اكتب الـ migration → اعرضها لي → انتظر موافقتي → نفّذ.

لا تشغّل migration على production أبداً بدون موافقة صريحة.
```

# 🔐 PROMPT 11: مراجعة أمنية

```
شغّل مراجعة أمنية شاملة لـ {{الموديول}}.

تحقق من:
1. SQL injection: كل queries عبر Prisma، لا raw SQL مع user input
2. XSS: كل user input يُعقّم قبل العرض
3. CSRF: كل POST/PUT/DELETE محمي
4. Authentication: كل endpoint محمي بـ guard مناسب
5. Authorization: permission check قبل كل عملية
6. RLS: tenant isolation مفعّل
7. Rate limiting: مطبّق على endpoints حساسة
8. Secrets: لا secrets في الكود أو الـ logs
9. File upload: validation + size limits + MIME check
10. Password hashing: Argon2id فقط

أعطني تقريراً مفصّلاً مع:
- ✅ ما هو سليم
- ⚠️ ما يحتاج تحسين
- 🚨 أي ثغرات حرجة

لا تصلح الثغرات الآن. أرسل التقرير أولاً.
```

# 💎 نصائح للحصول على أفضل النتائج

1. **كن محدداً جداً.** "ابني مهمة" ضعيف. "ابني Task 1.7 من TASKS.md حسب القواعد في CLAUDE.md" قوي.
2. **أعط Claude الوقت للتفكير.** اطلب منه شرح خطته قبل التنفيذ.
3. **لا تقبل بـ "تم" بدون اختبار.** اطلب screenshots أو نتائج tests.
4. **commit بانتظام.** كل sub-feature = commit.
5. **اطلب التوقف عند الشك.** أفضل أن يسأل Claude 100 سؤال من أن يخمّن واحداً.
6. **استخدم Antigravity Plan Mode.** للمهام الكبيرة، اطلب خطة قبل التنفيذ.
7. **راقب التكلفة.** $50 شهرياً معقول، إذا تجاوزت $200 فأنت تستخدم بشكل غير فعال.
8. **عدّل البرومتات.** هذه قوالب — حسّنها بناءً على تجربتك.
